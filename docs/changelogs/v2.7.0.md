# v2.7.0

> 发布日期：2026-04-16
> 版本范围：v2.6.0..HEAD

## 新功能

- `/req:fix` 新增 `--from-issue=#N` 参数，从 GitHub/Gitea issue 读取问题描述，分支名自动追加 `-iN` 后缀，完成后可选关闭 issue
- `/req:fix` 新增 `--auto` 非交互模式，跳过修复方案确认与 issue 关闭询问，修复完成后自动串联 `/req:commit` + `/req:pr`，实现一键修复 → 提交 → 发 PR
- 自然语言调度器扩展到四大类命令（修复与开发、状态流转、PR 操作），支持 Git 平台 issue/PR URL 识别，自动映射到 `/req:fix`、`/req:new`、`/req:review-pr` 等命令
- 自然语言调度器识别一键修复触发词（"一键修复"、"直接修并发 PR"、"自动修 #N" 等），映射到 `/req:fix --auto`

## 优化

- PreToolUse 确认 hook（`confirm-before-commit.sh` / `confirm-before-write.sh`）支持 `.claude/.req-auto` marker 放行机制：标记文件存在且 mtime 在 10 分钟内时直接 `exit 0`，让 `--auto` 模式无需手动放行 git commit 的原生确认对话框；TTL 保证异常退出残留标记不会长期生效
- `_common.md` 更新 issue 关联相关表格（分支命名、读取优先级、关闭策略），统一 `/req:new`、`/req:new-quick`、`/req:do`、`/req:fix` 四个命令的 issue 拉取与关联逻辑
- `.gitignore` 新增 `.claude/.req-auto` 标记文件忽略规则

## 文档

- `CLAUDE.md` 同步 `/req:fix` 新增参数说明，补充 auto 模式放行机制的钩子描述
