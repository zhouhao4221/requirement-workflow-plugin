# v2.17.0

> 发布日期：2026-04-24
> 版本范围：v2.16.0..HEAD

## 新增

- **Gitea 操作优先使用 `tea` CLI**：req 插件中所有调用 Gitea API 的命令新增 `tea` CLI 优先策略。本机存在 `tea` 且已 `tea login add` 当前 Gitea 实例 → 自动走 `tea`；否则回退既有 `curl + giteaToken` 路径，保持完全兼容。
  - 覆盖范围：`/req:issue`、`/req:pr`、`/req:done`、`/req:do`、`/req:fix`、`/req:release`、`/req:review-pr`
  - tea 覆盖：issue 增删改查/评论、PR 创建/列表/查看/合并/讨论评论、Release 主体创建
  - 仍走 curl：issue 评论列表、PR diff、PR 行内 review 评论、Release 资产上传、标签 CRUD（`tea` 无对应子命令或语义不一致）
  - **不会自动 `tea login add`**：避免把 token 写入 `~/.config/tea/config.yml` 这一全局副作用，需用户主动配置

## 文档

- 新增 `plugins/req/commands/_gitea_cli.md`：tea 检测脚本 + 操作覆盖矩阵 + 安全策略说明
- `_issue.md` 拉取逻辑示例补 tea 分支
- 8 个相关命令顶部统一加 CLI 优先级声明，指向新 snippet
- `CLAUDE.md` 在「Gitea 集成」段追加 CLI 优先策略说明
- 9 个命令的 `allowed-tools` 补 `tea:*`
