# v2.13.0

> 发布日期：2026-04-19
> 版本范围：v2.12.0..HEAD

## 行为变化

- **确认拦截默认关闭**（req 3.9.0）：req 插件的 PreToolUse 确认不再默认弹框，Write/Edit/Bash 全部直通。命令已通过多轮讨论 / 显式参数 / y/n 在自己的流程里完成意图确认，叠加一层 Hook 原生对话框被视为冗余打断。
  - 删除 `confirm-before-write.sh`：需求文档 / PRD / 模板 / changelog 的 Write/Edit 不再拦截
  - `confirm-before-commit.sh` 改为按 marker 文件 `.claude/.req-confirm-commit` 存在性决定是否拦截 git commit / mv / rm 需求文件
- **commit 确认改为自然语言开关**：想恢复"commit 前弹框"只需对 Claude 说"开启提交确认"。Claude 会：
  - `touch .claude/.req-confirm-commit`（marker 文件）
  - 在 auto-memory 中落一条 feedback 记录偏好
  - 关闭同理——"不用确认了" → `rm` marker + 更新 memory
  - **整个开关过程不再需要手动编辑 `settings.local.json` 或任何配置文件**
- `/req:release` 在 changelog 覆盖时的行为也一并解除拦截：若 `docs/changelogs/<version>.md` 已存在则直接覆盖（需重新生成时请先手动备份或删除）。

## Bug 修复

- **`confirm-before-commit.sh` 仓库根解析**（req 3.9.0）：修复从子目录执行 `git commit` 时 marker 检查错位的 P1 bug（Codex 在 PR #34 提出）。改用 `git -C "$CWD" rev-parse --show-toplevel` 解析仓库根再拼 marker 路径，非 git 目录 fallback 到 CWD。同修复顺带覆盖 `.claude/.req-auto`。

## 文档

- `natural-language-dispatcher` 技能新增 §七「确认偏好开关」：首个"直接动作（非命令映射）"类目，Claude 据此识别"开启/关闭/查询提交确认"意图。
- 三份 tutorial（zh / en / ko）的 `/req:fix --auto` 章节同步更新前置说明、表格、底层机制，反映新的 marker 驱动模型。
- `CLAUDE.md`、`_common.md` 的确认操作规范段落改写为"默认直通 + 可选开启"。
