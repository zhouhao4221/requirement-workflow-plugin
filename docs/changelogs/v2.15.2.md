# v2.15.2

> 发布日期：2026-04-22
> 版本范围：v2.15.1..HEAD

## 修复

- **修复 v2.15.1 引入的 1M context 付费墙问题**：v2.15.1 把 13 个命令显式降到 Sonnet 4.6，但未订阅 extra-usage 的用户在启用 1M 上下文的会话中运行这些命令时会直接报错 `Extra usage is required for 1M context`。本次按命令实际能力重新分配：
  - **能力足够改为 Haiku 4.5（9 个，不触发 1M 付费墙）**：`/req:commit`、`/req:review`、`/req:upgrade`、`/req:branch`、`/req:init`、`/req:migrate`、`/req:modules`、`/req:specs`、`/req:test_regression`
  - **复杂度偏高改为默认（5 个，跟随用户会话模型）**：`/req:edit`、`/req:issue`、`/req:new-quick`、`/req:pr`、`/req:split`
- **原因说明**：Haiku 全系仅 200K 变体，不存在 1M 付费墙；复杂命令留空 `model` 字段后跟随用户会话模型，由用户自行决定档位，不会被命令强行拉进 1M 模式。

## 文档

- `CLAUDE.md`「模型分级策略」章节从三档合并为两档：显式 Haiku / 不指定。删除「显式 Sonnet 的命令清单」，新增「为什么避免显式 Sonnet」说明（1M 付费墙）。
- 最终分布：Haiku 20 个、默认 14 个、Sonnet 0 个。
