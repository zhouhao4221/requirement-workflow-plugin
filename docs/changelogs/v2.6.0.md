# v2.6.0

> 发布日期：2026-04-16
> 版本范围：v2.5.0..HEAD

## 新功能

- 分支名编码 issue 编号：分支末尾追加 `-iN` 后缀（如 `feat/REQ-001-user-points-i12`、`fix/optimize-query-i42`），使 issue 关联可在跨会话场景下从分支名推断
- `/req:do --from-issue=#N` 完成后询问是否关闭关联 issue（gitea PATCH API / `gh issue close`），与 `/req:done` 行为对齐
- `/req:commit` 自动检测当前需求文档 `issue` 字段或当前分支名 `-iN` 后缀，在 commit message 末尾追加 `closes #N`，PR 合并时由 Git 平台自动关闭 issue

## 修复

- `/req:new --from-issue` 等命令拉取 Gitea issue 时，明确要求从 `branchStrategy.giteaUrl` 配置读取 API 地址，禁止从 git remote SSH 地址猜测 HTTPS URL（修复 SSH 端口与 HTTP 端口不一致时拉取失败的问题）

## 优化

- `_common.md` 新增「Issue 拉取规范」「Issue 与分支/提交的关联」两个章节，统一三个命令（`/req:new`、`/req:new-quick`、`/req:do`）的 issue 拉取与关联逻辑，避免重复描述
- `/req:dev` 创建分支时自动读取需求文档 `issue` 字段并追加 `-iN` 后缀
- `/req:done` 关闭 issue 时支持从分支名回退读取 issue 编号（文档优先）
