# v2.11.0

> 发布日期：2026-04-17
> 版本范围：v2.10.0..HEAD

## 需求分析能力增强（第一期）

对照主流需求分析方法论（IEEE 830 / Volere / BABOK / RUP / FURPS+ / INVEST / SMART / 5W2H / 5 Why）评审现有 `/req:new`、`/req:edit` 流程，落地三个高收益、低改动的补强项。

### 新增

- **REQ 模板第一章新增 1.5 范围与边界**：明确「本期包含 / 本期不做」，作为 scope creep 的第一道防线（对应 IEEE 830 Product Scope、Volere Scope of the Work）。
- **REQ 模板第一章新增 1.6 干系人**：角色 / 关注点 / 备注三列表格，覆盖提出方以外的业务、运营、合规、运维、客服等受影响方（对应 Volere / BABOK Stakeholder Map）。
- **REQ 模板第十章新增「假设」字段**：记录"相信为真但未验证的前提"，与「外部依赖」显式区分（假设 = 相信 X 是真的；依赖 = 必须拿到 Y），一旦不成立需重新评估方案。

### 行为变化

- **`requirement-analyzer` 第 1 轮提问升级**：质量门槛新增"识别至少一类干系人"；追问触发新增三条——
  - **5 Why 深挖**：用户描述的是解决方案而非问题时，追问「为什么需要？如果没有会怎样？」直到挖到真正的业务问题
  - **干系人识别**：用户仅提到提出方时，主动追问运营 / 客服 / 财务 / 合规 / 运维等受影响方
  - **范围边界**：场景描述发散时，追问「本期必须做的是哪些？哪些可以放到后面？」
- **阶段二一次性生成**：AI 同步填充新增的 1.5、1.6、假设三个字段，填充规则写入 `SKILL.md`。
- **兼容性**：旧需求文档不会被动改动；`/req:edit` 时由用户决定是否补齐新章节；`/req:update-template` 可一键同步新模板到项目本地，不覆盖需求文档。
