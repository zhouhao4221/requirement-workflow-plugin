# v2.3.1

> 发布日期：2026-04-12
> 版本范围：71b4818..HEAD（122 commits）

## 新功能

- **req 插件**
  - 新增 `/req:release` 颁布版本命令，支持合并 SQL、生成回滚脚本、打 tag、创建 Release
  - `/req:release` 支持从 develop 跨分支发布流程（git-flow 模式）
  - `/req:release` 支持 `--main` 参数临时覆盖主分支名
  - `/req:release` 自动推导版本号 + 流程守门增强
  - `/req:release` 合并 SQL 后自动删除原始文件
  - `/req:release` 完成后自动切回起始分支
  - 新增 `/req:do` 智能开发命令，AI 分析意图自动选择流程和分支前缀
  - 新增 `/req:fix` 轻量修复命令，AI 辅助定位 bug，无文档直接修复
  - 新增 `/req:pr` 命令，支持 Gitea API / gh CLI 创建 PR
  - 新增 `/req:review-pr` 命令，支持 AI 代码审查、平台评论和 PR 合并
  - 新增 `/req:specs` 规范文档共享命令，支持跨仓库共享数据类型和接口契约
  - 新增 `/req:split` 需求拆分分析命令
  - 新增 `/req:show` 命令，只读查看需求完整内容
  - 新增 `/req:branch` 分支管理命令及策略配置（GitHub Flow / Git Flow / Trunk-Based）
  - 新增 `/req:help` 使用教程命令
  - `/req:new` 需求分析支持多轮讨论，用户确认后再生成文档
  - `/req:edit` 支持多轮讨论、意图澄清和关联分析
  - `/req:dev` 自动创建和管理开发分支
  - `/req:commit` 自动切换需求分支，保护分支严格禁止直接提交
  - `/req:pr` 创建成功后提示是否删除本地分支
  - `/req:fix` 增加低成本关联需求匹配
  - 分支策略新增仓库类型配置（github/gitea/other），Gitea 集成 PR 创建
  - readonly 仓库 `/req:dev` 开发时不修改需求文档
  - 新增 PreToolUse Hook 实现原生确认对话框
  - 新增自然语言需求调度技能
  - 命令 frontmatter 补齐 + SessionStart hook + hook 超时修复
  - Memory 隔离规则：模板化命令和 skill 禁止受 auto-memory 影响

- **pm 插件**
  - 新增项目管理助手插件，支持周报/月报/统计/方案/风险/站会等多场景输出

- **api 插件**
  - 新增 API 对接插件，支持 Swagger 解析、字段映射、代码生成
  - `api:gen` 增加字段变更分析和关联页面影响扫描

## 修复

- `/req:release` tag 必须打在 mainBranch 上
- `/req:release` 跨分支流程修复 PR 复用 bug
- `/req:commit` 重写分支保护逻辑，保护分支检查提升为第一步
- `/req:commit` 保护分支上从改动文件和上下文推断需求
- `/req:do` 直接在当前分支开发，不创建新分支
- `/req:fix` 调整流程顺序，先分析 bug 再建分支，避免空分支残留
- hook 脚本仅对 docs/requirements 目录下的文件触发，非需求文件静默跳过
- plugin.json 显式声明 hooks 路径，修复缓存同步不触发
- 移除 plugin.json 中重复的 hooks 声明
- 清理残留的 GITEA_TOKEN 引用，统一改为 giteaToken
- giteaToken 改为直接从配置文件读取，不再依赖环境变量
- 完成需求时必须同步缓存归档

## 重构

- 工具集更名为 DevFlow，仓库迁移至 devflow-claude

- 精简 REQ 模板，分离需求定义与实现方案
- 移除所有模板文档中的 emoji
