# v2.9.0

> 发布日期：2026-04-16
> 版本范围：v2.8.0..HEAD

## 行为变化

- **`/req:review-pr review` 上传评审评论前默认需要确认**（req 3.5.0）：AI 代码审查完成后，先展示将要提交到 PR 的精简版 Markdown 预览，等待用户 `y/n` 确认后再上传。避免审查结果在用户没机会检查的情况下直接对外发布。
- 用户回复自由文本时视为修改意见，AI 按意见调整精简版内容后重新预览。
- `repoType = "other"` 不变，仅本地展示，不触发确认。

## 新功能

- **`/req:review-pr review --auto`**：跳过上传确认，直接提交精简版评论。与 `/req:fix --auto` 保持一致的 `--auto` 参数体系。
- **自然语言调度器扩展**：识别一键审查类触发词（"一键审查"、"自动审查"、"审查并提交"、"审完直接评论"、"别问我"）并映射到 `/req:review-pr review --auto`，识别时回复说明能力边界。

## 优化

- 用户拒绝上传时，反馈明确指向 `/req:review-pr review --auto` 作为补救路径。
- `--auto` 只影响 `review` 子命令的上传确认。`/req:review-pr merge` 的分支清理询问由 `branchStrategy.deleteBranchAfterMerge` 控制；`/req:review-pr fetch-comments` 的"是否应用修改"询问保留，避免 AI 误改代码。

## 文档

- 三份 tutorial 新增 13.3 章节「其他支持 `--auto` 的命令」，说明 `/req:review-pr review --auto` 的默认行为变化和触发方式（中文 / 英文 / 韩文）。
- `plugins/req/skills/natural-language-dispatcher/SKILL.md` 的 §4.3 补充 `--auto` 触发词清单和示例。
