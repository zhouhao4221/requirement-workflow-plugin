# 生产诊断

## 概述

生产诊断模块提供一套 Claude Code 插件，帮助开发者通过自然语言对生产环境进行**只读诊断**和**修复建议**，覆盖应用日志、错误堆栈、应用运行态等场景。

模块核心边界：

- **只读**：所有生产侧操作只允许查看、读取，禁止任何写/改/删
- **建议而非执行**：AI 输出文字诊断报告和修复建议，不自动改代码、不自动改生产
- **风控前置**：通过 Claude Code Hook 在工具调用前拦截危险命令
- **审计可追溯**：所有远程操作落 JSONL 审计日志

与 `claude-safe-ops` 互补：`claude-safe-ops` 负责"执行类"运维动作，本模块负责"诊断类"只读分析。

| 编号 | 描述 | 状态 | 完成日期 |
|------|------|------|----------|

| F2 | 生产服务配置管理（服务清单、主机白名单、日志路径） | 规划中 | - |

## 业务规则

| 规则 | 说明 |
|------|------|
| 只读边界 | 所有远程操作只允许只读命令（tail/head/cat/grep/awk/sed/less/wc/find -name/ls/ps/df/free/uptime 等） |
| 主机白名单 | SSH 目标主机必须在配置文件中显式登记，不在白名单直接拒绝 |
| 禁止文件写入 | 禁止管道重定向到文件（`> / >> / tee`） |
| 禁止写类命令 | 阻断 rm / mv / cp / chmod / chown / kill / systemctl / service / docker / kubectl 的写操作 |
| 建议不执行 | 修复建议以纯文本给出，禁止自动生成 patch 或 Edit 代码 |
| 审计全量 | 所有 SSH 命令、拉取的日志片段、产出建议全部落 JSONL 审计 |
| Hook 防绕过 | 每会话首次调用校验 Hook 完整性，缺失则告警 |
| 配置驱动 | 堆栈解析不写死语言/框架，交由 AI 自适应识别 |
