## 3. 业务规则

### 3.1 用户认证

- 小程序端调用 `wx.login()` 获取 code
- 后端用 code 换取 openid 和 session_key
- 首次登录自动注册用户
- 返回 JWT token 用于后续请求认证

### 3.2 记账规则

- 金额必须大于 0，最多两位小数
- 必须选择分类
- 日期默认当天，可选择历史日期
- 备注可选，最长 200 字符

### 3.3 分类规则

- 分类分为「收入」和「支出」两种类型
- 系统预设默认分类，用户可自定义
- 分类不可删除（只能禁用），防止历史数据丢失

### 3.4 统计规则

- 按日统计：当日各分类收支汇总
- 按周统计：本周每日收支趋势
- 按月统计：本月每日收支趋势 + 分类占比

## 5. API 设计

### 5.1 用户认证

#### 小程序登录
```
POST /api/v1/auth/wx-login
Request:
{
    "code": "wx.login返回的code"
}
Response:
{
    "token": "jwt_token",
    "user": {
        "id": 1,
        "nickname": "用户昵称",
        "avatar_url": "头像URL"
    }
}
```

### 5.2 记账管理

#### 创建记账
```
POST /api/v1/records
Request:
{
    "category_id": 1,
    "amount": 99.50,
    "type": 1,
    "record_date": "2026-01-09",
    "remark": "午餐"
}
```

#### 查询记账列表
```
GET /api/v1/records?start_date=2026-01-01&end_date=2026-01-31&page=1&page_size=20
```

#### 删除记账
```
DELETE /api/v1/records/:id
```

### 5.3 分类管理

#### 获取分类列表
```
GET /api/v1/categories?type=1
```

#### 创建自定义分类
```
POST /api/v1/categories
Request:
{
    "name": "宠物",
    "type": 1,
    "icon": "pet"
}
```

### 5.4 统计报表

#### 获取统计数据
```
GET /api/v1/statistics?period=month&date=2026-01
Response:
{
    "total_income": 10000.00,
    "total_expense": 5000.00,
    "balance": 5000.00,
    "daily_trend": [...],
    "category_stats": [...]
}
```
