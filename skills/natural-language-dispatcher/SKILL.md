# 自然语言需求调度器

当用户用自然语言描述需求操作时，解析意图并执行对应的命令流程。

---

## 触发条件（必须同时满足）

### 新增需求

用户消息中包含以下**任一关键词**：
- "新增需求"、"新建需求"、"创建需求"

**不要求**提供需求编号，编号由系统自动生成。

### 修改需求

用户消息中**同时包含**：
1. 关键词："修改需求"、"编辑需求"、"变更需求"
2. 需求编号：`REQ-XXX` 或纯数字编号（如 `025`、`3`）

**两个条件缺一不可**，仅说"修改需求"但不带编号时，不触发本技能。

---

## 意图解析规则

从用户消息中提取以下信息：

| 字段 | 解析方式 | 示例 |
|------|---------|------|
| 操作类型 | 新增 / 修改 / 变更 | "新增需求" → 新增，"变更需求" → 修改 |
| 需求编号 | `REQ-XXX` 或纯数字，补齐为 `REQ-XXX` 格式 | "025" → REQ-025 |
| 需求标题 | 新增时从消息中提取功能描述 | "新增需求 用户积分管理" → 标题="用户积分管理" |
| 修改内容 | 修改时提取用户想改的部分 | "增加导出功能" → 编辑意图 |
| 需求类型 | 如有提及前端/后端/全栈则提取 | "新增后端需求" → type=后端 |
| 所属模块 | 如有提及模块名则提取 | "用户模块的需求" → module=用户模块 |

---

## 执行流程

### 新增需求

解析完成后，按 `/req:new` 命令的完整流程执行：

1. 读取 `commands/new.md` 中定义的执行流程
2. 将解析出的标题、类型、模块作为参数传入
3. 后续流程与 `/req:new` 完全一致（生成编号 → 收集信息 → 创建文档）

**示例映射：**
- "新增需求 用户积分管理" → `/req:new 用户积分管理`
- "新建一个后端需求，做订单导出" → `/req:new 订单导出 --type=后端`
- "创建需求 会员等级，属于用户模块" → `/req:new 会员等级 --module=用户模块`

### 修改需求

解析完成后，按 `/req:edit` 命令的完整流程执行：

1. 读取 `commands/edit.md` 中定义的执行流程
2. 将解析出的需求编号作为参数传入
3. 将用户描述的修改内容作为编辑意图，在进入编辑流程后直接定位到相关章节
4. 后续流程与 `/req:edit` 完全一致

**示例映射：**
- "修改025需求，增加导出功能" → `/req:edit REQ-025`，编辑意图="增加导出功能"
- "编辑需求 REQ-003，补充业务规则" → `/req:edit REQ-003`，编辑意图="补充业务规则"
- "修改需求3，把接口改成批量操作" → `/req:edit REQ-003`，编辑意图="把接口改成批量操作"
- "变更需求 REQ-003，调整业务规则" → `/req:edit REQ-003`，编辑意图="调整业务规则"

---

## 不触发的情况

以下情况**不应触发**本技能，避免误匹配：

- 仅提到"需求"但无"新增/新建/创建/修改/编辑/变更"动词：如"看一下025需求"
- "修改需求"/"变更需求"但没有编号：如"修改需求的模板"
- 讨论需求概念而非操作：如"这个需求太大了需要拆分"
- 涉及其他命令的操作：如"开始开发025"、"提交025需求评审"
