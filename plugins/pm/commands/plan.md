# 生成方案文档

根据当前项目数据和用户指定的主题，AI 生成方案文档。

## 命令格式

```
/pm:plan <主题> [--save]
```

**参数说明：**
- `<主题>`：**必填**，方案主题（如「Q2 排期方案」「技术架构升级」「人力资源评估」）
- `--save`：可选，直接保存不询问

**示例：**
- `/pm:plan Q2排期方案` — 基于待办需求生成排期
- `/pm:plan 技术债务清理` — 基于代码热点和修复频率
- `/pm:plan 新人上手指南` — 基于项目结构和模块信息

---

## 执行流程

### 1. 采集相关数据

根据主题关键词自动判断需要哪些数据：

| 主题关键词 | 采集数据 |
|-----------|---------|
| 排期、计划、迭代 | 活跃需求 + 历史完成速度 + 贡献者 |
| 技术、架构、重构 | 代码结构 + 热点文件 + Git 变更 |
| 资源、人力、团队 | 贡献者统计 + 需求分布 |
| 风险、复盘、回顾 | 风险检测 + 历史数据 |
| 上手、培训、介绍 | PRD + 模块 + 项目结构 |

### 2. AI 生成方案

**生成要求：**
- 方案必须基于实际项目数据，不凭空设想
- 引用具体的需求编号、文件路径、统计数据
- 包含可执行的行动项
- 区分「建议」和「结论」

### 3. 输出格式

```
═══════════════════════════════════════════════
方案文档：<主题>
═══════════════════════════════════════════════
项目：<project> | 生成日期：YYYY-MM-DD
───────────────────────────────────────────────

## 背景

（AI 基于项目数据分析当前状况）

## 目标

（从主题推导出要解决的问题）

## 方案详情

（具体方案内容，引用项目数据）

## 行动项

- [ ] 行动 1（负责人/时间）
- [ ] 行动 2
- [ ] 行动 3

## 风险与依赖

（方案执行的前置条件和风险）

═══════════════════════════════════════════════
*由 /pm:plan 自动生成 · YYYY-MM-DD*
```
