# 项目总进度

从全局视角展示所有需求的进度、里程碑节点和交付时间线。适合给上级看项目全貌。

## 命令格式

```
/pm:progress [--save]
```

---

## 执行流程

### 1. 采集数据

```python
reqs = collect_requirements()
prd = collect_prd()
modules = collect_modules()
git_tags = git_tags()  # 版本里程碑
```

### 2. 生成进度视图

按以下维度组织：

**2.1 总体进度**
- 需求完成率（已完成/总数）
- PRD 完善度
- 版本发布历史（Tag 列表）

**2.2 按模块进度**
- 每个模块下的需求及其状态
- 模块完成度百分比

**2.3 需求时间线**
- 按创建时间排列所有需求
- 标注每个需求的当前状态和关键日期
- 标注里程碑 Tag 的位置

**2.4 甘特视图（文字版）**

```
需求时间线
──────────────────────────────────────────────
REQ-001 用户积分    80%  开发中
REQ-002 积分前端    50%  开发中
REQ-003 订单导出   100%  已完成
REQ-004 消息通知    30%  测试中
REQ-005 支付对账     0%  草稿
──────────────────────────────────────────────
整体进度：3/5 完成（60%）
```
